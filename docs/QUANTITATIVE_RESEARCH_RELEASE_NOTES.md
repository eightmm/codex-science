# Codex Science 0.5 quantitative research release

This release turns the existing mathematics, statistics, numerical-analysis, dimensional-analysis, uncertainty, and formal-proof guidance into executable, hash-covered research contracts.

## Added

- preregistered quantitative research-design audit;
- experimental-unit-aware independent and paired two-group inference;
- exact or seeded Monte Carlo randomization tests and deterministic bootstrap intervals;
- bounded mathematical expression evaluation and counterexample search;
- mathematical claim statuses, proof-obligation graphs, finite-exhaustion receipts, and formal-kernel receipts;
- bounded Lean proof preview and execution in an existing workspace;
- numerical refinement, convergence-order, residual, invariant, precision, tolerance, and cross-method verification;
- SI dimensional equation checking and bounded unit conversion;
- covariance-aware linear and seeded Monte Carlo uncertainty propagation;
- quantitative sidecar validation and claim-level cross-review;
- an end-to-end deterministic quantitative acceptance bundle;
- DeepMind-style progressive references and machine-readable L3/L4 maturity declarations for the core quantitative skills.

## Evidence boundaries

- A p-value is not practical importance, replication, or causality.
- A bounded computation is not a general proof.
- Exact finite exhaustion proves only the exact declared finite domain.
- Kernel checking covers an exact formal statement; informal/formal correspondence remains a separate review.
- Numerical convergence on a finite refinement range does not prove asymptotic convergence or model validity.
- Dimensional consistency is necessary but not sufficient for a correct physical equation.
- Uncertainty propagation is conditional on its input distributions and covariance model.
- The checked acceptance fixture validates software and research contracts rather than an empirical scientific conclusion.

## Validation gate

The release candidate gate executes release, connector, reference, native-skill maturity, inventory, wrapper, model, reviewer, SBDD, quantitative acceptance, artifact-hash, and claim-level review checks before a managed update can activate the release.
